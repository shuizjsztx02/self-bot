# Notion 项目管理系统创建指南

## 概述
本指南将帮助您在 Notion 工作区创建一个完整的项目管理系统，包含数据库创建、任务管理、查询功能等核心功能。

## 第一步：创建项目数据库

### 1.1 创建新页面
1. 在 Notion 侧边栏点击 **+ New Page**
2. 命名为 **"项目管理"** 或 **"Projects"**
3. 选择 **Table** 视图

### 1.2 添加项目属性
| 属性名 | 类型 | 说明 |
|--------|------|------|
| 项目名称 | Title | 项目名称（默认） |
| 状态 | Select | 进行中/已完成/暂停 |
| 优先级 | Select | 高/中/低 |
| 负责人 | Person | 项目负责人 |
| 开始日期 | Date | 项目开始时间 |
| 截止日期 | Date | 项目截止时间 |
| 进度 | Number | 0-100% |
| 描述 | Text | 项目描述 |
| 标签 | Multi-select | 项目分类标签 |

### 1.3 创建示例项目
添加几个示例项目：
- 网站改版项目
- 产品发布项目
- 市场推广项目

## 第二步：创建任务数据库

### 2.1 创建任务数据库
1. 在项目管理页面内创建新数据库
2. 命名为 **"任务管理"** 或 **"Tasks"**
3. 选择 **Table** 视图

### 2.2 添加任务属性
| 属性名 | 类型 | 说明 |
|--------|------|------|
| 任务名称 | Title | 任务名称（默认） |
| 所属项目 | Relation | 关联到项目数据库 |
| 状态 | Select | 待办/进行中/已完成 |
| 优先级 | Select | P1/P2/P3/P4 |
| 负责人 | Person | 任务负责人 |
| 截止日期 | Date | 任务截止时间 |
| 预计工时 | Number | 小时数 |
| 实际工时 | Number | 实际花费小时 |
| 标签 | Multi-select | 任务类型标签 |
| 描述 | Text | 任务详细描述 |

### 2.3 配置数据库关系
1. 在 **"所属项目"** 属性中，选择 **Relation**
2. 关联到 **"项目管理"** 数据库
3. 启用双向关系

## 第三步：创建查询视图

### 3.1 按项目查看任务
1. 在任务数据库顶部点击 **+ Add a view**
2. 选择 **Gallery** 或 **Board** 视图
3. 命名为 **"按项目分组"**
4. 分组依据选择 **"所属项目"**

### 3.2 按状态查看任务
1. 添加新视图 **Board**
2. 命名为 **"看板视图"**
3. 分组依据选择 **"状态"**
4. 列：待办/进行中/已完成

### 3.3 按优先级筛选
1. 添加新视图 **Table**
2. 命名为 **"高优先级任务"**
3. 添加筛选器：`优先级` 包含 `P1`

### 3.4 按时间筛选
1. 添加新视图 **Calendar**
2. 命名为 **"任务日历"**
3. 日期属性选择 **"截止日期"**

## 第四步：创建仪表板

### 4.1 创建主仪表板页面
1. 创建新页面 **"项目仪表板"**
2. 使用 **/database** 命令嵌入数据库

## 第七步：查询数据库的方法

### 7.1 基本查询
```javascript
// 查询所有进行中的项目
filter(prop("状态") == "进行中")

// 查询我的任务
filter(prop("负责人") == currentUser())

// 查询本周到期的任务
filter(prop("截止日期") >= startOfWeek(now()) and prop("截止日期") <= endOfWeek(now()))
```

### 7.2 复杂查询
```javascript
// 查询高优先级且延迟的任务
filter(
  and(
    prop("优先级") == "P1",
    prop("截止日期") < now(),
    prop("状态") != "已完成"
  )
)

// 按项目统计任务数量
group(prop("所属项目"), count())
```

## 第九步：故障排除

### 常见问题
1. **数据库关系不显示**
   - 检查双向关系设置
   - 确认数据库在同一工作区

2. **查询结果不正确**
   - 检查筛选条件
   - 确认属性类型匹配

3. **性能问题**
   - 减少数据库中的行数
   - 优化视图数量
